Verify Smokeview

  • Update Linux and Mac repositories to the latest revision. (Use Smartsvn to update the Windows repository)
  • Run test cases Run FDS cases used to test smokeview (allow several hours to complete). This step does not need to be performed every time.
  • Create figures Run Smokeview scripts to generate figures for the User and Verification Guides.
  • Generate the Smokeview User and Verification Guides
  • View the Smokeview Users Guide
  • View the Smokeview Verification Guide

Build, Package and Upload Smokeview

Note: Windows 32 and 64 bit versions of Smokeview are built using MS Visual Studio (each on their respective platforms)
  • Get the SVN revision number of the Smokeview source directory.
  • Increment the version number (SMVVERSION found in options.h . Add a dummy comment in main.c to force a repository update.
  • IMPORTANT - Commit changes so that the proper revision number appears when Smokeview runs.
  • Rebuild a Windows release version of Smokeview.
  • Regenerate test cases and Smokeview user's guide using instructions above under Verify Smokeview. Rebuilding the User's guide ensures that the proper Smokeview version information appears in the guide.
  • Build Smokediff - Linux, OSX,
  • Copy Smokediff to Smokeview bin directory - Linux, OSX,
  • Build Smokezip - Linux, OSX,
  • Copy Smokezip to Smokeview bin directory - Linux, OSX,
  • Build fds2ascii - 32 bit Windows, 64 bit Windows, Linux, OSX,
