Verify Smokeview

  • Update all repositories (Windows, Linux, Mac) to the latest revision..
  • Run test cases Run FDS cases used to test smokeview (allow several hours to complete). This step does not need to be performed every time.
  • Create figures Run Smokeview scripts to generate figures for the User's Guide
  • Generate User's Guide Use pdflatex and bibtex to generate the User's guide
  • View User's Guide Examine the User's guide to verify that Smokeview is working properly.

Package and Upload Smokeview

  • Get revision number Get the SVN revision number of the Smokeview source directory.
  • Increment the version number (SMVVERSION found in options.h . Add a dummy comment in main.c to force a repository update.
  • IMPORTANT - Commit changes so that the proper revision number appears when Smokeview runs.
  • Rebuild a Windows release version of Smokeview.
  • Regenerate test cases and Smokeview user's guide using instructions above under Verify Smokeview. Rebuilding the User's guide ensures that the proper Smokeview version information appears in the guide.
